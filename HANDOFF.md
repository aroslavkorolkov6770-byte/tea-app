## Состав реализованного релиза

### Сервер

- `app/api/ai-content-draft/route.ts` - администраторский multipart API, лимиты файлов, запрос Alice AI, разбор и проверка JSON-черновика.
- `app/lib/aliceAi.ts` - общий серверный клиент Alice AI и извлечение текста ответа.
- `app/lib/documentTextExtractor.ts` - извлечение текста из PDF, DOCX, таблиц, презентаций, OpenDocument, RTF и текстовых файлов.
- `package.json`, `package-lock.json` - добавлены `mammoth`, `pdf-parse`, `jszip`.

### Интерфейс

- `app/tasks/components/AiContentGenerator.tsx` - двухшаговое окно загрузки и настройки, отдельные светлые и темные стили.
- `app/tasks/components/Education.tsx` - меню ручного/AI-создания и перенос черновика в существующие редакторы темы и теста.
- AI не сохраняет материал автоматически. Сохранение выполняется только кнопкой администратора после проверки.

### Документация

- `AGENTS.md` требует обновлять `PROJECT_CONTEXT.md`, `TASKS.md` и `HANDOFF.md` после каждого завершенного изменения кода.
- Текущий контекст и статус AI-генератора записаны в Markdown-файлы проекта.

## Безопасная заливка при будущей команде пользователя

```powershell
git status --short
git diff --check
git add <только файлы кода>
git commit -m "Короткое понятное описание"
git push
```

После GitHub на сервере выполняется обычное обновление кода, зависимостей, сборка и перезапуск процесса. Точный путь текущего развертывания: `/var/www/html`. Последовательность на сервере: `cd /var/www/html`, затем `git pull --ff-only`, `npm install` при изменении зависимостей, `npm run build`, `pm2 restart tea-hub`. Перед этим нужно проверить, что команды не затронут `data/` и другие пользовательские файлы. Никогда не очищать данные и не выполнять разрушительные команды Git.
